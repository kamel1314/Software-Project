# Campus Events System — Scrum Meetings

This document summarizes the Scrum meetings conducted during the development of the Campus Events System project as part of the **CSCI 313 – Software Engineering** course.

The project development started on **16 October 2025** and continued until **20 December 2025**. Scrum meetings were held regularly to plan work, review progress, and prepare the project for submission.

---

## Scrum Meeting 1 — Project Planning

**Date:** 16 October 2025  
**Type:** Sprint Planning  

### Discussed Topics
- Understanding project requirements
- Defining system actors (Student, Admin)
- Identifying core features and project scope
- Planning initial tasks and responsibilities

### Decisions
- Use a web-based client–server architecture
- Implement separate roles for students and admins
- Use GitHub for version control and collaboration

### Action Items
- Set up repository structure
- Begin backend and frontend implementation
- Create initial backlog

---

## Scrum Meeting 2 — Core Development Review

**Date:** 30 October 2025  
**Type:** Sprint Review  

### Discussed Topics
- Backend API implementation progress
- Frontend integration with backend
- Event registration functionality
- Admin event management features

### Issues Identified
- Merge conflicts between branches
- Incomplete system documentation

### Decisions
- Resolve merge conflicts and stabilize main branch
- Continue feature implementation

### Action Items
- Fix Git issues
- Complete remaining backend endpoints
- Improve frontend integration

---

## Scrum Meeting 3 — Feature Completion

**Date:** 13 November 2025  
**Type:** Sprint Review  

### Discussed Topics
- Completion of student and admin features
- Use case diagram updates
- Initial testing of system functionality

### Issues Identified
- Missing use cases in UML diagram
- Minor frontend inconsistencies

### Decisions
- Update use case diagram to match implementation
- List UI improvements as future enhancements

### Action Items
- Finalize use case diagram
- Improve frontend stability
- Begin preparing documentation

---

## Scrum Meeting 4 — Testing and CI/CD

**Date:** 27 November 2025  
**Type:** Sprint Review / Retrospective  

### Discussed Topics
- Backend testing and validation
- CI/CD pipeline setup using GitHub Actions
- Security and dependency checks
- Deployment approach

### Issues Identified
- Unclear distinction between Docker and local deployment
- Docker setup not required for current development

### Decisions
- Use local deployment as the official setup
- Treat Docker support as an optional future enhancement
- Improve deployment documentation clarity

### Action Items
- Finalize CI/CD configuration
- Update deployment documentation
- Prepare project backlog and issues log

---

## Scrum Meeting 5 — Documentation and Integration

**Date:** 11 December 2025  
**Type:** Sprint Review  

### Discussed Topics
- Review of README, Deployment, Backlog, and Issues files
- Scrum board status and task completion
- System integration review

### Decisions
- Documentation considered complete
- Core system functionality stable
- Remaining tasks marked as future improvements

### Action Items
- Perform final consistency checks
- Prepare project for final submission

---

## Scrum Meeting 6 — Final Review and Submission

**Date:** 20 December 2025  
**Type:** Final Review  

### Discussed Topics
- Final system status
- Frontend and backend stability
- Submission readiness

### Decisions
- System approved for submission
- UI and cloud deployment listed as future enhancements
- Project officially completed

---

## Summary

- Scrum meetings were conducted regularly from October to December 2025
- Development progress was reviewed incrementally
- Issues were identified and resolved during each sprint
- Tasks were tracked using a Scrum board in GitHub Projects
- The project followed basic Scrum practices suitable for an academic software engineering course
